## Decisions

- Phase 4는 기존 UI를 전면 재설계하지 않고 사용자 용어와 정보 우선순위를 정리하는 단계로 한정한다.
- 옛 Issue의 요구는 원격 Issue 번호가 아니라 Phase 0 보고서에 보존된 충돌 목록을 근거로 다시 승인한다.
- 승인 전 진행을 막는 대신 각 충돌 항목에 권장안을 제시하고, 별도 변경 요청이 없으면 권장안으로 구현할 수 있게 했다.
- 사용자 지정 `review` 상태와 시스템 무결성 검사를 분리하고, 허용된 미등록 값은 자동 오류로 만들지 않는 방향을 기본안으로 정했다.
- 메모는 회원을 기억하는 실사용 단서이므로 개인정보 우려보다 검색성과 식별 편의를 우선하되, 검색 순위는 가장 낮게 두기로 했다.
- 국가명 반복으로 카드가 복잡해지는 것을 막기 위해 한국어 국가는 입력 화면에서만 사용하고 카드에서는 `KR`, `BR`, `MX`, `XX` 코드만 표시하기로 했다.
- 메모 미리보기는 원본 데이터를 변경하지 않고 공백만 정리한 뒤 10글자를 넘을 때 말줄임표를 붙이도록 결정했다.
- 닉네임은 한국어 두 글자 이상을 포함하면 영문·숫자·공백을 함께 사용할 수 있도록 검증 규칙을 완화했다. 한국어가 전혀 없거나 제어 문자가 포함된 값은 계속 거부한다.
- 환경변수 예시 파일을 저장소에 유지하지 않고, local-only 파일을 직접 생성하도록 문서를 정리했다. 실제 secret이 Git에 들어갈 예외 규칙도 제거했다.
- Phase 5 자동 gate는 실제 production 권한을 변경하지 않도록 SQL 정의와 읽기 전용 감사 쿼리를 분리했다. DNS·OAuth·write smoke test는 승인 후 수동 실행한다.
