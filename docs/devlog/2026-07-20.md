- 배포 workflow에 `verify:deploy`를 추가해 production build와 Phase 5 정적 배포 안전 검사만 배포 gate로 사용하도록 분리했다. 전체 Phase 1–4 회귀 검사는 기존 `verify:phase5` 명령으로 보존하되, 작은 문구·CSS 수정의 Pages 배포를 오래된 테스트 기대값이 막지 않도록 했다.
- production build와 TypeScript 검사는 통과했다.

## Decisions

- 기능에 영향을 주지 않는 설명 문구만 제거하고, 국가 선택지·닉네임 검증·관리 메뉴 동작은 유지했다.
- 공통 메뉴 컴포넌트 전체에서 설명을 삭제하지 않고 선택적 prop으로 만들어 기존 설명이 필요한 메뉴와 설명이 없는 메뉴를 함께 지원한다.
- 설정 메뉴는 참고 프로젝트와 같은 native `details` 방식으로 구현하고, 관리·로그아웃 실행 시 메뉴를 닫도록 했다.
- 자주 쓰는 `회원 추가`는 설정 메뉴에 넣지 않고 상단에 남겨 접근성을 유지했다. 관리 기능과 세션 종료만 설정 메뉴로 이동했다.
- 홈 아이콘은 소스 폴더가 아닌 Vite의 정적 자산 폴더에서 제공해 production 경로와 일치시키기로 했다.
- Pages 자동 배포는 배포에 직접 필요한 build·정적 보안 검사를 통과하면 진행하고, 전체 회귀 테스트는 별도 수동 검증으로 분리하기로 했다. 현재 배포 실패는 기능 오류가 아니라 테스트 계약의 문구 순서 불일치였기 때문이다.
